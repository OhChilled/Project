# Інструкція з оновлення проєкту (Update & Rollback)

## 1. Підготовка до оновлення

Перед початком оновлення необхідно:

- переконатися, що програма не запущена
- завершити всі активні екземпляри застосунку
- перевірити, що поточна версія працює коректно

---

## 2. Створення резервних копій

Перед оновленням потрібно створити резервні копії:

### Виконуваний файл
copy build\untitled2.exe build\untitled2_backup.exe

---

## 3. Перевірка сумісності

Перед оновленням потрібно перевірити:

- чи не змінилися вимоги до CMake
- чи не потрібна новіша версія компілятора
- чи не змінилася логіка роботи програми
- чи нова версія сумісна зі старими результатами

---

## 4. Планування часу простою

Оскільки проєкт є консольним застосунком, час простою мінімальний.

Оновлення виконується між запуском програми.

---

## 5. Процес оновлення

### 5.1 Оновлення коду
git pull

### 5.2 Перезбірка проєкту
cd build
cmake ..
cmake --build .

---

## 6. Зупинка служб

Окремі служби у проєкті не використовуються.

Перед оновленням достатньо:
- закрити програму
- переконатися, що вона не запущена

---

## 7. Розгортання нового коду

Після виконання git pull та повторної збірки нова версія застосунку вважається розгорнутою.

---

## 8. Міграція даних

Міграція даних не потрібна, оскільки проєкт не використовує СУБД.

---

## 9. Оновлення конфігурацій

Конфігураційні файли у проєкті не використовуються.

---

## 10. Перевірка після оновлення

Після оновлення потрібно:

Запустити програму:

Windows:
.\untitled2.exe

Linux:
./untitled2

Перевірити:
- програма запускається
- не виникає помилок
- результати обчислень коректні

---

## 11. Критерії успішного оновлення

Оновлення вважається успішним, якщо:

- код оновлено без конфліктів
- збірка завершилась без помилок
- програма запускається
- результати коректні

---

## 12. Процедура відкату (rollback)

У разі помилки потрібно:

### Варіант 1 — через Git

git log --oneline
git checkout "назва працюющого відкату"

cd build
cmake ..
cmake --build .

---

### Варіант 2 — через резервну копію

del build\untitled2.exe
move /Y build\untitled2_backup.exe build\untitled2.exe
