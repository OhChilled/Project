# Інструкція з розгортання проєкту у production-середовищі

## 1. Загальна інформація
Проєкт є консольним застосунком мовою C++, призначеним для моделювання процесу зближення та стикування космічних апаратів.

Проєкт не використовує:
- вебсервер
- application server
- СУБД
- кешуючі сервіси

Розгортання полягає у збиранні та запуску застосунку на цільовій машині.

---

## 2. Архітектура проєкту
Структура застосунку включає:
- модуль запуску програми
- модулі математичного моделювання
- модуль тестування
- конфігурацію збірки CMake

Схема роботи:

Користувач  
↓  
Консольний застосунок  
↓  
Обробка вхідних параметрів  
↓  
Математичне моделювання  
↓  
Формування результатів  
↓  
Вивід результатів у консоль або файл

---

## 3. Вимоги до апаратного забезпечення
Мінімальні вимоги:
- Архітектура: x86_64
- CPU: 2 ядра
- Оперативна пам’ять: 4 GB RAM
- Вільне місце на диску: 1 GB

Рекомендовані вимоги:
- CPU: 4 ядра
- Оперативна пам’ять: 8 GB RAM
- SSD для швидшої збірки та запуску

---

## 4. Необхідне програмне забезпечення
На цільовій машині повинно бути встановлено:
- Git
- CMake 3.20 або новіше
- C++ компілятор з підтримкою C++17

### Linux
Рекомендовано:
- Ubuntu 22.04+ або інший сучасний дистрибутив
- g++

### Windows
Рекомендовано:
- Windows 10/11
- Visual Studio Build Tools або MinGW

---

## 5. Налаштування мережі
Для базової роботи проєкту спеціальні мережеві налаштування не потрібні, оскільки проєкт є локальним консольним застосунком.

Підключення до мережі потрібне лише для:
- клонування репозиторію
- отримання оновлень
- завантаження залежностей, якщо це потрібно

---

## 6. Конфігурація серверів / цільової машини
На цільовій машині необхідно:
1. Створити робочий каталог для проєкту
2. Клонувати репозиторій
3. Створити каталог `build`
4. Виконати конфігурацію та збірку

---

## Архітектура проєкту

Проєкт є консольним застосунком і не використовує вебсервер, application server, СУБД або кешування.

Схема роботи:

Користувач
↓
Консольний застосунок
↓
Модуль обробки вхідних параметрів
↓
Модуль математичного моделювання
↓
Обчислення траєкторії та перевірка умови стикування
↓
Вивід результатів у консоль або файл

## Вимоги до середовища розробки

Для запуску та розробки проєкту потрібні:

* Git
* CMake 3.20 або новіше
* C++ компілятор з підтримкою C++17
* IDE або редактор коду:

    * CLion, або
    * Visual Studio / VS Code, або
    * будь-який інший редактор з підтримкою C++

## Підготовка середовища

### Windows

1. Встановити Git
2. Встановити CMake
3. Встановити компілятор C++:

    * MinGW, або
    * Visual Studio Build Tools, або
    * MSVC через Visual Studio

### Linux

1. Встановити Git
2. Встановити CMake
3. Встановити g++

Наприклад для Ubuntu:

```
sudo apt update
sudo apt install git cmake g++
```

## Клонування репозиторію

```
git clone https://github.com/OhChilled/Project.git
cd untitled2
```

## Збірка проєкту

```
mkdir build
cd build
cmake ..
cmake --build .
```

## Запуск проєкту

Після успішної збірки виконуваний файл буде знаходитися в каталозі `build`.

### Linux

```
./untitled2
```

### Windows

```
.\untitled2.exe
```

## Запуск у режимі розробки

1. Внести зміни у вихідний код
2. Перезібрати проєкт:

```
cmake --build .
```

3. Повторно запустити виконуваний файл

## Робота з тестами
