# 📚 Генерація документації

У проєкті використовується **Doxygen** для автоматичної генерації документації з коду C++.

## 🔧 Встановлення

Необхідно встановити Doxygen та перевірити доступність команди:

```bash
doxygen --version
```

## ▶️ Генерація документації

У кореневій директорії проєкту виконати:

```bash
doxygen Doxyfile
```

## 📂 Результат

Після виконання команди буде створено папку:

```bash
docs_output/html
```

Головний файл документації:

```bash
docs_output/html/index.html
```

Відкрийте його у браузері для перегляду.

## 🧩 Що документується

У проєкті документуються всі публічні інтерфейси, зокрема:

структури (Params, State)
функції (fx, fy, calcR, shouldDock, rk4Step)
параметри функцій (@param)
значення, що повертаються (@return)

## ✍️ Стиль документації

Використовується формат Doxygen:

@brief — короткий опис
@param — опис параметрів
@return — опис результату

## 🔄 Оновлення документації

Документацію необхідно підтримувати в актуальному стані при зміні:

- сигнатур функцій
- алгоритмів
- структури проєкту

Це забезпечує зрозумілість коду та спрощує його підтримку.
